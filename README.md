# BookMemory

BookMemory is a Vue 3 + TypeScript web app to manage a personal reading library, track reading sessions, and monitor progress and stats.

## Stack

- Vue 3 (`<script setup>`) + Vite
- TypeScript
- Pinia
- Vue Router
- Vue I18n (ES/EN)
- Tailwind CSS
- Firebase (Auth + Firestore)
- Vitest + Vue Test Utils
- PWA (`vite-plugin-pwa`)

## Main Features

- Authentication with route guards (`/login` public-only, app routes protected)
- Library management (`/books`, `/books/:id`)
- Reading sessions and progress tracking (`/reading`)
- Stats dashboard (`/stats`)
- Sync center and offline queue conflict handling (`/sync`)
- Global notifications and confirm modals
- Footer with dynamic app version (`VITE_APP_VERSION`)

## Environment Variables

Copy `.env.example` to `.env` and configure:

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
VITE_GOOGLE_BOOKS_API_KEY=
VITE_APP_VERSION=
```

Notes:
- In local/dev you can set `VITE_APP_VERSION=v0.0.0-dev` (or similar).
- In CI deploy, `VITE_APP_VERSION` is generated automatically from workflow version logic.

## Available Scripts

- `npm run dev`: start development server
- `npm run build`: typecheck + production build (`dist/`)
- `npm run preview`: preview production build
- `npm run typecheck`: TypeScript checks (`vue-tsc --noEmit`)
- `npm run lint`: ESLint strict mode (`--max-warnings=0`)
- `npm run lint:fix`: ESLint auto-fix
- `npm run test`: Vitest watch mode
- `npm run test:run`: Vitest single run
- `npm run smoke:mvp`: lint + typecheck + tests + build

## Testing

- Unit/integration tests are co-located (examples: `src/stores/*.test.ts`, `src/modules/*/views/*.test.ts`, `src/services/*.test.ts`).
- Default test environment: `jsdom` (configured in `vite.config.ts`).

## CI/CD (GitHub Actions)

### Quality workflow

File: `.github/workflows/lint.yml`

Runs on:
- Push to `main` and `develop`
- Pull request to `main` and `develop`

Matrix tasks:
- `lint`
- `typecheck`
- `test:run`
- `build`

### Release + Deploy workflow

File: `.github/workflows/deploy.yml`

Triggers:
- PR closed on `main` (only executes release path when merged)
- Push to `main`
- Manual dispatch

Behavior:
- Always gates deploy with `npm run smoke:mvp`.
- Semantic version bump on merged PRs to `main`:
  - PR title contains `(major)` -> major bump
  - PR title contains `(minor)` -> minor bump
  - default -> patch bump
- Pushes version commit/tag when bump applies.
- Builds app and deploys `dist/` via FTP (`SamKirkland/FTP-Deploy-Action`).
- Avoids duplicate deploy for the push event generated by the same merged PR.

Required deployment secrets:
- `FTP_SERVER`
- `FTP_USERNAME`
- `FTP_PASSWORD`
- `FTP_DIR`
